Configuring PowerView

If PowerView is to be deployed on a Raspberry Pi consider configuring PowerView before installation. It might be easier.

PowerView is configured through the file: PowerView.exe.config (XML format). Therefore to configure PowerView open the PowerView.exe.config file in a text editor.

Default Configuration

The PowerView default configuration assumes execution on a Raspberry Pi Linux environment. The default configuration assumes deploymet in Denmark.

Configuring Services

The PowerView services are configured in the following section of the configuration file:

	<Service>
		<BaseUrl value="http://localhost:47362" />
	</Service>

The BaseUrl element defines the URL at which PowerView provides its web (HTTP) services. Visualizing readings in a browser can be initiated by navigating to this base URL.

Configuring Database

Aspects of the PowerView reading database is configured in the following section of the configuration file:

	<Database>
		<Name value="/var/lib/PowerView/PowerView.sqlite3" />
		<Backup>
			<MinimumIntervalDays value="14" />
			<MaximumCount value="5" />
		</Backup>
	</Database>

The Name element defines the path and file name for the PowerView database. The Backup element defines database backup parameters. Whenever PowerView starts a database backup is performed depending the values of these parameters. Database backups are placed in a DbBackup sub-directory inside the directory declared in the Name element.
